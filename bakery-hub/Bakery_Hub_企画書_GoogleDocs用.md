# Bakery Hub 企画書

**高槻市の個人経営パン屋向け 業務管理SaaS**
作成日：2026年6月7日

---

## 1. ターゲット

**メインターゲット：高槻市内の個人経営パン屋（従業員1〜10名）**

| 項目 | 内容 |
|------|------|
| 店舗規模 | 従業員1〜10名 |
| 月商 | 250〜600万円 |
| オーナー月収 | 平均30万円（利益は薄い） |
| ITリテラシー | スマホ・LINE・Instagramは使える。ExcelやPCツールは苦手な層が多い |

**ペルソナ：田中さん（44歳・女性・ベーカリー経営8年目）**

- パートスタッフ2〜3名で切り盛り。製造は自分が中心
- 予約注文はLINE・ノート・口頭の3ルートに分散
- 毎朝の予約確認に15分、予約忘れ・焼きすぎが月に数回発生
- スマホとLINEは使えるが、Excelは苦手

## 2. 想定課題

パン屋の現場で繰り返し起きている6つの共通課題。

| # | 課題 | なぜ起きるか |
|---|------|------------|
| 1 | 予約管理の分散 | 「とりあえず連絡できる手段」で受けてしまう |
| 2 | 顧客情報が蓄積されない | 記録の「ひと手間」が営業中は後回しになる |
| 3 | 廃棄ロスの最適化不足 | 販売数・売切れ時間を記録していない |
| 4 | SNS依存の業務負荷 | 集客ツールが業務管理を兼ねている矛盾 |
| 5 | データ分析が感覚依存 | 記録の仕組みがなく分析の土台がない |
| 6 | 店長の時間ロス | 兼任ゆえの業務中断が製造・接客の時間を削る |

**核心：** パン屋の利益は薄い。パンは日持ちせず、売れ残りは廃棄になる。
製造数のズレと業務中断は、この薄い利益を直接削る。
**Bakery Hub は「利益防衛ツール」として位置づける。**

---

## 3. 解決方法

「予約・顧客・商品・在庫」を1つのクラウドに集約し、分散と属人化を解消する。

| 業務 | 導入前 | Bakery Hub 導入後 |
|------|--------|-----------------|
| 予約管理 | 電話・DM・メモに分散 | 1画面に集約、ステータスで一目把握 |
| 製造数決定 | カン頼み | 予約数＋定番販売数を自動集計 |
| 顧客対応 | オーナーの記憶頼み | 顧客カルテで誰でも同じ対応 |
| 在庫・廃棄 | 記録なし | 販売数・廃棄数を日次記録、分析に活用 |
| スタッフ共有 | 口頭・紙で漏れる | クラウドでリアルタイム共有 |
| 問い合わせ対応 | 営業中に都度中断 | 在庫表示・予約フォームで中断を削減 |

**なぜExcelやGoogleフォームでは解決しないのか：**
予約・在庫・顧客・分析が「連動」しないため。
Bakery Hub は予約→在庫→ステータスが自動で連動する点が本質的に異なる。

## 5. 差別化ポイント

| 観点 | 既存ツール（Airレジ・スマレジ等） | Bakery Hub |
|------|-------------------------------|-----------|
| 軸足 | POS・会計 | 予約・顧客・在庫の業務管理 |
| 予約・取り置き | 弱い／非対応 | 中核機能 |
| 顧客カルテ | 限定的 | 常連の購入傾向・メモを蓄積 |
| 業種特化 | 汎用 | パン屋の業務フローに最適化 |
| UI | 多機能ゆえ複雑 | IT苦手層向けにシンプル |

**3つの差別化メッセージ**

1. **パン屋専用** — 汎用POSではなく、パン屋の「売切れ・取り置き・予約」に最適化
2. **時間も守る** — モノのロスだけでなく、店長の業務中断（時間のロス）も減らす
3. **すぐ使える** — IT苦手な店長でも初日から使えるシンプルさ

## 6. 月額いくらなら売れると思うか

### 想定価格

| プラン | 月額 | 対象 |
|-------|------|------|
| スタータープラン | **9,800円** | 1店舗・利用者3名まで |
| スタンダードプラン | **19,800円** | 1店舗・利用者10名まで＋分析レポート |

### この価格を選ぶ理由

オーナーの月収は平均30万円と薄利。月1〜2万円は安くない出費になる。
ただし以下の効果を実感できれば、「利益防衛の投資」として成立する。

- 廃棄削減（例：毎月5,000円分のパンが廃棄 → 半減で2,500円の利益改善）
- 予約ミス防止（1件のクレーム・返金リスクの回避）
- 店長の時間創出（月10時間の業務中断を削減）

**飲食店向けSaaSの相場（Airレジ・Square等）が月数千〜数万円帯であることを参考に設定。**
ただし、ベーカリー特化サービスでの支払意欲は未調査のため仮説。
**実店舗インタビュー5〜10件で必ず検証が必要。**

### 検証すべき仮説

1. 個人パン屋は月1〜2万円を払ってでも予約・在庫管理を解決したいか
2. アナログ運用の店長が、ツールへの入力を習慣化できるか（3カ月無料トライアルで継続率を計測）
