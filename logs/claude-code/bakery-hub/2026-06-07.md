## やったこと

### 1. 原価計算本実装（/analytics/profit-preview/page.tsx）
- モックデータから実データへ全面書き直し
- Supabase から products / recipe_items / ingredients / inventory_logs を取得
- 原価/個 = レシピの(usage_quantity x purchase_price)の合計で算出
- 商品別に 販売数・売上・原価計・粗利・利益率を表示
- レシピ未設定商品は「レシピ未設定」と表示
- 利益率50%未満の商品はアラートセクションで表示
- 日付範囲フィルタ付き

### 2. CSV出力機能
- src/components/CsvDownloadButton.tsx: 汎用CSVダウンロードボタン（BOM付き・日本語Excel対応）
- src/app/(app)/analytics/SalesCsvButton.tsx: 日別売上CSV
- src/app/(app)/staff/WorkHoursCsvButton.tsx: 勤務時間CSV（名前・メール・権限・出勤日数・合計時間）
- analytics/page.tsx と staff/page.tsx に組み込み済み

### 3. シフトスマホ対応（src/app/(app)/staff/ShiftCalendar.tsx）

- hidden/md:block の切り替えで実装

## 設計判断

- CSV の BOM 付き出力: Windows Excel での文字化け防止のため採用
- 原価計算はレシピベース（recipe_items x ingredients.purchase_price）で算出。レシピ未設定商品は計算対象外とし「未設定」表示にした
- シフト表のスマホ対応は CSS hidden/md:block で既存テーブルを温存し、スマホ用カードビューを追加する方式（既存ロジックへの影響最小化）
