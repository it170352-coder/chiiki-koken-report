# Claude Code 棚卸し v1

以下を**質問せず・順番に・全部自動で**実行してください。完了したら最後にサマリレポートを出力すること。「分からなかった」「権限が足りなかった」項目は『未確認』として明示してOKです（推測で埋めない）。

---

## 棚卸しの目的
- 自分が今「何を持っていて」「何を使っていて」「何が放置されているか」を一覧化
- 整理候補（削除・統合・更新が必要なもの）を抽出
- 次の改善アクションを3つ以内に絞る

---

## Step 1: グローバル設定の棚卸し

### 1-1. `~/.claude/` 配下の構成

以下を実行して中身を一覧化：

```bash
ls -la ~/.claude/
ls -la ~/.claude/agents/ 2>/dev/null
ls -la ~/.claude/projects/ 2>/dev/null
ls -la ~/.claude/skills/ 2>/dev/null
```

確認項目：
- グローバル `CLAUDE.md` の有無・最終更新日・行数
- `settings.json` / `settings.local.json` の有無
- `agents/` 配下のエージェント数とファイル名
- `projects/` 配下のメモリディレクトリ数

### 1-2. グローバル `CLAUDE.md` の内容把握

`~/.claude/CLAUDE.md` を読み込んで、以下を抽出：
- 「言語」「認証・課金」「セキュリティ」「ツール実行許可」などの章構成
- 最終更新からの経過日数

### 1-3. settings.json の確認

- permissions (allow/deny ルール) の件数
- hooks の設定有無
- 環境変数の設定状況（`ANTHROPIC_API_KEY` 等）

---

## Step 2: プロジェクト資産の棚卸し

### 2-1. ローカルプロジェクト一覧

作業ディレクトリ（推定: `~/Desktop/<会社名>/` または相当する場所）を確認し、以下を表形式で：

| プロジェクト名 | 最終更新 | Git リモート | 本番URL有無 | ステータス推定 |
|---|---|---|---|---|

ステータス推定の判断軸：
- 直近1週間以内に更新あり → **稼働中**
- 1ヶ月以内に更新あり → **進行中**
- 1〜3ヶ月停止 → **休眠**
- 3ヶ月以上停止 → **廃止候補**

### 2-2. 各プロジェクトの CLAUDE.md 充実度

各プロジェクト直下に `CLAUDE.md` があるか確認し、最低記載事項（役割 / 本番URL / 完成度 / 技術スタック / 触ってはいけないファイル）が揃っているかチェック。

---

## Step 3: メモリ・ナレッジの棚卸し

### 3-1. メモリインデックスの確認

`~/.claude/projects/<work-dir>/memory/MEMORY.md` を読み込んで：
- エントリ数（行数）
- 「次回再開メモ」の最終更新日
- リンク先ファイルが実在するか（壊れたリンクの検出）

### 3-2. メモリと実態の整合確認（F-2 対策）

各メモリの「STEP X 完了」「本番URL: ...」と書かれている項目について、以下のいずれかで外形確認：
- 本番URL を curl / WebFetch で叩いて HTTP ステータス確認
- Supabase MCP が使えるなら `list_migrations` で適用状態確認
- Vercel CLI が使えるなら `vercel ls` で稼働確認

齟齬があったら「⚠️ メモリと実態に乖離あり」として列挙。

### 3-3. ナレッジ蓄積場所の棚卸し

- Obsidian Vault（`~/Desktop/Obsidian/` または相当）の存在確認
- `logs/claude-code/` 配下のセッションログ数
- 直近1ヶ月のログ作成頻度

---

## Step 4: エージェント基盤の棚卸し

`.claude/agents/` 配下の各エージェントについて：

| エージェント名 | モデル | 最終更新 | 役割（1行） | 使用形跡 |
|---|---|---|---|---|

使用形跡の判断軸：
- セッションログに名前が登場する → **使用中**
- 1ヶ月以上ログに登場しない → **未使用候補**
- 役割が他エージェントと重複している → **統合候補**

---

## Step 5: MCP / スキル / フックの棚卸し

### 5-1. MCP サーバー接続状況

```bash
claude mcp list 2>/dev/null
```

接続されている MCP サーバー名と数を確認。

### 5-2. 利用可能なスキル
セッション開始時に表示されるスキル一覧から、以下を分類：
- ビルトインスキル
- カスタムスキル（自作・チーム共有）
- プラグイン由来のスキル

### 5-3. hooks 設定
`settings.json` の `hooks` セクションがあれば内容を確認。自動化されている動作を列挙。

---

## Step 6: 運用パターンの棚卸し

### 6-1. 直近1ヶ月のセッションログ傾向

`~/.claude/projects/` 配下の各プロジェクトの jsonl ファイルから、直近1ヶ月で：
- 最も触ったプロジェクト Top3
- セッション数
- ログ未記録の日数（連続）

### 6-2. 繰り返している失敗パターン

セッションログ内に「失敗」「問題」「事故」セクションがある場合、過去30日で同じ失敗が2回以上出現していないか確認。
